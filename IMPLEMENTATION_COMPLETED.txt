MODIFICATION #1: LECTURE ENGAGEMENT REGISTER WITH ABSENT ROLL NUMBERS
================================================================================

REQUIREMENT:
Store complete lecture engagement data including absent roll numbers

IMPLEMENTATION:
✅ Modified lecture_engagement table - Added: absent_roll_numbers TEXT[] column
✅ Enhanced Faculty Dashboard - Lecture Engagement tab now:
   - Retrieves attendance records for the specific date/subject/faculty
   - Displays list of absent students (by roll number)
   - Stores absent roll numbers in database
   
DATA STORED:
✓ day/date                     (date field)
✓ name of faculty              (from users.name via faculty_id)
✓ subject name                 (from subjects.name via subject_id)
✓ topic covered                (topic_covered field)
✓ lecture number               (lecture_number field)
✓ % syllabus covered           (syllabus_percent field)
✓ total student present        (total_present - calculated from attendance)
✓ only absent roll numbers     (absent_roll_numbers - NEW TEXT[] array)

DATABASE TABLE:
lecture_engagement
├─ id (SERIAL PRIMARY KEY)
├─ faculty_id (INTEGER FK)
├─ subject_id (INTEGER FK)
├─ date (DATE)
├─ topic_covered (TEXT)
├─ lecture_number (INTEGER)
├─ syllabus_percent (DECIMAL)
├─ total_present (INTEGER)
├─ total_absent (INTEGER)
└─ absent_roll_numbers (TEXT[]) ← NEW COLUMN

ADMIN ENHANCEMENT:
✅ Download Engagement tab now includes absent_roll_numbers column in CSV

HOW TO USE:
1. Faculty logs in
2. Goes to Dashboard → Lecture Engagement tab
3. Selects Subject and Date
4. System automatically retrieves attendance records
5. Displays: "Absent Students: 101, 103, 105"
6. Faculty enters topic and lecture details
7. Clicks "Submit Engagement"
8. Absent roll numbers are stored in database

MODIFICATION #2: FACULTY FILE MANAGEMENT (ASSIGNMENTS & NOTES)
================================================================================

REQUIREMENT:
Faculty can upload and delete assignments/notes from database

IMPLEMENTATION:
✅ Created faculty_resources table with BYTEA binary storage
✅ Added Upload Resources tab to Faculty Dashboard
✅ Added View Resources tab to Faculty Dashboard

DATABASE TABLE:
faculty_resources
├─ id (SERIAL PRIMARY KEY)
├─ faculty_id (INTEGER FK)
├─ subject_id (INTEGER FK)
├─ file_name (VARCHAR 255)
├─ file_type (VARCHAR 50) - extracted from filename
├─ file_data (BYTEA) ← BINARY FILE STORAGE IN DATABASE
├─ resource_type (VARCHAR 50) - Assignment/Notes/Other
└─ uploaded_date (TIMESTAMP) - auto-generated

FILE STORAGE METHOD:
✓ Binary data stored directly in PostgreSQL/NeonDB (BYTEA format)
✓ No external file system required
✓ Automatic metadata tracking (filename, type, upload date)
✓ Faculty can upload any file type

DATABASE FUNCTIONS ADDED:
• upload_resource(faculty_id, subject_id, file_name, file_data, resource_type)
• get_faculty_resources(faculty_id, subject_id=None)
• delete_resource(resource_id)
• get_resource_file(resource_id)

UI FEATURES:
Feature: Upload Resources (New Tab in Faculty Dashboard)
├─ Select Subject dropdown
├─ Resource Type selector (Assignment / Notes / Other)
├─ File uploader
└─ Submit button

Feature: View Resources (New Tab in Faculty Dashboard)
├─ Display table with columns:
│  ├─ File name
│  ├─ Resource type
│  ├─ Subject name
│  └─ Upload date/time
└─ Delete button for each resource

HOW TO USE:
1. Faculty logs in
2. Goes to Dashboard → Upload Resources tab
3. Selects Subject: "Mathematics (SY)"
4. Selects Resource Type: "Assignment"
5. Uploads file: "Assignment1.pdf"
6. Clicks Upload
7. File stored as binary in database
8. Can view in View Resources tab
9. Can delete anytime

MODIFICATION #3: ADMIN STUDENT MANAGEMENT (UPDATE & DELETE)
================================================================================

REQUIREMENT:
Admin can update student details and delete students from database

IMPLEMENTATION:
✅ Created Manage Students tab in Admin Dashboard
✅ Full CRUD operations for student records
✅ Cascade delete of attendance records

DATABASE FUNCTIONS ADDED:
• get_all_students()
• update_student(student_id, roll_no, name, class_name)
• delete_student(student_id) - with cascade delete of attendance

UI FEATURES:
Feature: Student List (New Admin Tab: Manage Students)
├─ Display table showing all students:
│  ├─ Roll No
│  ├─ Name
│  ├─ Class
│  ├─ Edit button
│  └─ Delete button
│
└─ Edit Form (appears when Edit clicked)
   ├─ Roll No (editable)
   ├─ Name (editable)
   ├─ Class (dropdown selector)
   ├─ Save Changes button
   └─ Cancel option

Delete Functionality:
├─ Delete button triggers confirmation dialog
├─ Warning: "All related attendance records will be deleted"
├─ Confirm Delete button
├─ Cancel button
└─ On confirmation:
   ├─ All attendance records deleted (CASCADE)
   └─ Student record deleted

HOW TO USE - EDIT:
1. Admin logs in
2. Goes to Admin Dashboard → Manage Students tab
3. Sees list of all students
4. Clicks Edit button for a student
5. Modifies: Roll No, Name, or Class
6. Clicks Save Changes
7. Changes persist in database

HOW TO USE - DELETE:
1. Admin logs in
2. Goes to Admin Dashboard → Manage Students tab
3. Clicks Delete button for a student
4. Confirmation dialog appears with warning
5. Clicks Confirm Delete
6. Student and all attendance records deleted

CASCADE DELETE BEHAVIOR:
├─ Student table: record removed
├─ Attendance table: all records for that student removed
└─ Preserves data integrity
